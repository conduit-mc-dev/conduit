# 编码规范合规检查

对最近的代码变更进行编码规范合规审查。使用 `git diff HEAD~1` （或用户指定的范围）获取变更内容，然后逐条核对以下规范来源中的规则。

## 规范来源

1. 全局用户指令：`~/.claude/CLAUDE.md`（重点关注「Git 提交规范」和「代码注释规范」章节）
2. 项目编码准则：`$ARGUMENTS` 中若指定了文件则读取该文件，否则读取 `CLAUDE.md`（重点关注「编码准则」§1-§6）

读取以上文件后，提取所有可验证的规则作为检查项。

## 输出格式

按以下格式输出每个发现：

```
[违规/建议/通过] §{维度编号} — {文件名}:{行号范围}
{简要描述}
```

最后给出总结：共检查 N 个文件，发现 X 个违规、Y 个建议、Z 个通过。

如果全部通过，输出：✅ 全部检查通过，变更符合编码规范。
